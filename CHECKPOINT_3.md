## 1. Программа и методика тестирования

Для верификации криптографического ядра и инфраструктуры логирования был проведен комплекс ручных интеграционных тестов в изолированном контейнере.

### Тест-кейс 1: Успешное шифрование и изоляция секретов
* **Входные данные:** Файл `secrets.env` с содержимым `API_TOKEN=super_secret_value`. В системном окружении установлен `SECRETGUARD_MASTER_KEY=ProdKey32Bytes...`
* **Команда выполнения:** `dotnet run -- encrypt --input secrets.env --output secrets.env.enc`
* **Результат:** Сгенерирован бинарный файл `secrets.env.enc`. В логах `logs/info.log` зафиксирована успешная операция. Исходный текст в файле полностью зашифрован.

### Тест-кейс 2: Обработка исключений и проверка целостности логов
* **Входные данные:** Запрос на дешифрацию поврежденного или несуществующего файла.
* **Команда выполнения:** `dotnet run -- decrypt --input non_existent_file.enc`
* **Результат:** Программа не упала с критическим сбоем процесса. В лог-файл ошибок `logs/error.log` записано структурированное сообщение: `[ERROR] Критическая ошибка: Зашифрованный файл non_existent_file.enc не существует.`

## 2. Результаты развертывания и Docker-инфраструктура

Сборка Docker-образа выполнена успешно на базе многоэтапного `Dockerfile`. Логи поставляются в stdout контейнера и дублируются на хост-машину через монтирование директорий (Volumes).

### Инструкция по сборке и запуску контейнера:

1. **Сборка образа утилиты:**
```bash
docker build -t secretguard:latest .
```

2. **Запуск шифрования внутри контейнера с прокидыванием секретов и папки логов:**
```bash
docker run --rm \
  -e SECRETGUARD_MASTER_KEY="MyDockerSecretKey32BytesLong!!" \
  -v "$(pwd)/data:/app/data" \
  -v "$(pwd)/logs:/app/logs" \
  secretguard:latest encrypt --input data/target.env --output data/target.env.enc
```
